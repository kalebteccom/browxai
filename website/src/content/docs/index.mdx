<div class="browx-section not-content">
  <h2 class="browx-section-title">Why it exists</h2>
  <p class="browx-section-sub">
    browxai is not a wrapper over someone else's MCP server. It owns its own
    Playwright and CDP transport so it can own the whole session lifecycle.
  </p>

  <div class="browx-grid">
    <div class="browx-card">
      <span class="browx-card-mark">// surface</span>
      <h3>Token-efficient</h3>
      <p>
        <code>snapshot()</code> is a scoped, paginated, budgeted accessibility
        tree. <code>find()</code> returns ranked candidates with evidence, not a
        guess. Results stay small on purpose.
      </p>
    </div>
    <div class="browx-card">
      <span class="browx-card-mark">// safety</span>
      <h3>Safe by default</h3>
      <p>
        Capability-gated tools, an origin allow and block list, confirmation
        hooks, and a hard anti-wedge deadline on every call. Arbitrary JS, full
        response bodies, the OS clipboard, and network mocking are all off until
        you opt in.
      </p>
    </div>
    <div class="browx-card">
      <span class="browx-card-mark">// clients</span>
      <h3>Model-agnostic</h3>
      <p>
        Any MCP client over stdio. Use it from Claude, Codex, or anything else
        that speaks the protocol. It is not tied to one model or one vendor.
      </p>
    </div>
    <div class="browx-card">
      <span class="browx-card-mark">// sessions</span>
      <h3>Sessions and lifecycle</h3>
      <p>
        Isolated per-session contexts: persistent, incognito, or attach to an
        existing Chrome you already control. Headed and headless. Config is
        driven over MCP, with no out-of-band setup.
      </p>
    </div>
  </div>
</div>
